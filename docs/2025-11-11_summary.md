# 2025-11-11 작업 요약

- **서버 스위처 UX 개편**: 사이드바 상단을 `Server` 개념으로 정리하고 드롭다운에서 서버 전환/생성이 가능하도록 구현. 서버 생성 시 자동으로 `# general` 채널과 섹션 세트가 붙는다.
- **Topbar 재배치**: 검색창 및 `Help / Create / Notifications / Settings` 버튼을 우측 끝으로 이동해 글로벌 컨트롤을 한눈에 볼 수 있게 정리.
- **워크시트 상세 뷰 강화**: 구글 시트 스타일의 셀 선택/범위 하이라이트/방향키 이동/Shift 드래깅을 추가하고, 복사-붙여넣기·포뮬러 바·자동완성 함수(`SUM/AVERAGE/MIN/MAX`)와 행/열 자동 맞춤, 더블클릭 편집까지 지원하도록 구현.

> 다음 단계: 서버 전환 시 각 모듈 스토어 상태 초기화 전략과, 포뮬러 함수/키보드 단축키 확장 고려 필요.

- **Dashboard 헤더 조정**: “워크스페이스 허브” 카드를 화이트 배경으로 정리하고 텍스트와 버튼 크기를 조절해 상단 공간을 보다 꽉 채웠습니다.
- **quick action 집중**: 다가오는 일정·팀 펄스 블록을 제거하고 quick action만 남겨 헤더를 더욱 심플하게 구성했습니다.
- **사이드바 워크스페이스 플로우**: 설정 버튼에서 이름/아이콘 변경과 삭제 확인 UI를 추가하고 `/setup/workspace` 마법사로 새 워크스페이스를 연결했습니다.

- **워크스페이스 위저드 이미지 타입 안정화**: 이미지 업로드 후 FileReader 결과를 문자열인지 먼저 확인하고 상태를 갱신하여 image 필드 타입 오류를 제거함.
- **사이드바 워크스페이스 흐름 정리**: 현재 워크스페이스를 activeWorkspace로 명명하고 관련 훅/설정/삭제 로직에서 일관적으로 참조하여 가독성과 타입 안정성을 높임.
- **컴포넌트 구조 통일**: `views` 아래 산재하던 각 도메인 뷰를 `components/calendar`, `components/chat/view`, `components/dashboard`, `components/docs`, `components/issues`, `components/worksheet`, `components/settings` 등으로 재배치하고, 상위 레이아웃(Topbar/Sidebar)과 오른쪽 패널도 `components/layout`/`components/issues`에 정리하여 전체 폴더 구조 일관성을 확보함.
