# 2025-11-10 작업 정리

## 오늘 처리한 내용
- `components/views/issues/KanbanView.tsx`를 CalendarView 구조에 맞춰 재구성했습니다. 상단에 `KanbanHeader`를 두어 요약 통계와 탭을 묶고, 본문은 단일 스크롤 섹션으로 정리해 레이아웃 일관성과 확장성을 확보했습니다.
- Planning 테이블 UI를 단순화했습니다. 업무명은 `truncate` 처리로 길이를 제한했고, 수량 열을 담당자 열로 교체했으며, 진행률 슬라이더·타임라인 최소 폭·시작/종료일 표기를 조정해 좁은 공간에서도 정보가 명확히 보이도록 했습니다. 테이블 상세 행과 빈 상태 `colSpan`도 새 구조에 맞춰 수정했습니다.
- 워크플로 목데이터(`mocks/kanban.ts`)의 제목·소유자·ID를 짧고 이해하기 쉬운 한글 표현(`wf-1`, `wf-1-a` 등)으로 통일했습니다. 테이블에는 내부 ID를 더 이상 노출하지 않아 실사용 화면이 깔끔해졌습니다.
- 변경 이후 `npm run build`를 반복 실행해 타입/린트/빌드가 모두 통과함을 확인했습니다.
