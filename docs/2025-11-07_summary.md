# 2025-11-07 작업 정리

## 오늘 처리한 내용
- `store/issues.ts`에 있던 모든 목데이터(워크플로 노드, JobSheet/페인트/서브컨트랙트, 설비, 인력, 알림 등)를 `@mocks/kanban.ts`로 분리하고 `tsconfig.json`에 `@mocks/*` 경로를 등록해 관리 편의성을 높였습니다.
- KanbanView의 워크플로 섹션에서 잘못된 템플릿 리터럴로 인해 발생하던 JSX 구문 오류를 수정했고, reference 예제 컴포넌트(GanttChart, JobPainting)의 타입 누락도 보완했습니다.
- 실시간 상태/알림 로직이 새 구조와 호환되도록 zustand 스토어 타입을 다듬고, `npm run build`가 정상적으로 통과하는지 확인했습니다.
- Issues 화면에서 RightPanel을 제거하고, KanbanView의 Production 섹션을 WBS 테이블+간트 차트 2열 구성으로 전환했습니다. 각 행은 `Details` 버튼으로 인라인 상세를 펼쳐 배정/의존관계·상태/진척률 편집 UI(드롭다운+슬라이더)를 제공하며, 변경 내용은 실시간 브로드캐스트(`emitWorkflowUpdate`)로 반영됩니다. 헤더 통계와 Production orders 카드의 폭도 확대해 가독성을 높였습니다.
